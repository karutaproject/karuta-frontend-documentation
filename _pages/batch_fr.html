Fonctionnement

Ensemble de commandes de script permettant d'automatiser la création, modification et suppression de portfolios, ainsi que la création ou suppression d'usagers.

À partir d'un ou plusieurs modèles de portfolios, d'un script batch et d'un fichier CSV, KARUTA crée les usagers et leurs portfolios. Le fichier CSV contient les données relatives à chaque usager à créer. Le script de batch indique les opérations à effectuer : créer l'usager, créer son ou ses instances de portfolios à partir de modèles de portfolio, partager les nouveaux portfolios avec les usagers dans des rôles spécifiés, remplir les données de l'usager dans les portfolios, etc.

Exemple de script batch

Ce script correspond à l'algorithme suivant :

Créer le projet pour la cohorte
Pour chaque ligne du fichier CSV
 Créer le compte pour l'étudiant si il n'existe pas
 Créer le portfolio de l'étudiant
 Mettre à jour le champs Nom dans le portfolio
 Partager le portfolio avec l'étudiant dans le rôle "etudiant"

Exemple de fichier CSV

Le fichier CSV contient les informations nécessaires à l'exécution du batch.

Format du fichier CSV :

  • Ligne 1 Les noms des variables globales. Les variables globales sont les variables qui ont les même valeurs pour chaque usager à créer (par exemple, les codes des modèles de portfolio à instancier, un code de programme ou une session). Dans le script de batch, les références aux variables globales sont précédées de //.
    model_code en cellule A1 est un mot réservé : c'est le code du portfolio contenant le script de batch à exécuter.
  • Ligne 2 Les valeurs des variables globales. Dans l'exemple présenté la valeur de model_code est exemple.batch-creer-cohorte : c'est le code du portfolio contenant le script de batch.
  • Ligne 3 Les noms des variables locales. Les variables locales sont les variables qui ont des valeurs différentes pour chaque usager à créer (par exemple, le nom de l'étudiant, son prénom, courriel, etc).
  • Lignes 4 et + Les valeurs des variables locales, habituellement un usager par ligne.

NOTE IMPORTANTE SUR LA SYNTAXE DES NOMS DE VARIABLES
Les noms des variables globales et locales ne peuvent PAS inclure les caractères suivants : trait d'union (-), plus (+), astérisque (*), barre oblique (/), pourcentage (%) ou dièse (#).
Le caractère de soulignement (_) est permis.

Pour chaque ligne

ForEachLine est une boucle et est généralement le point de départ d'un script de batch. Les commandes à l'intérieur de l'élément ForEachLine seront exécutées pour chaque ligne du fichier CSV (lignes 4+).

Pour chaque arbre

Si Alors Sinon

Créer utilisateur

Créer un usager. Ne pas oublier de remplacer les valeurs par défaut de cet élément par vos propres noms de variables locales (ligne 3 du fichier CSV).

Champs à renseigner :

  • Identifiant Identifiant unique du l'usager qui servira de login dans le système Karuta (par exemple un matricule ou courriel).
  • Prénom Prénom de l'usager.
  • Nom Nom de famille de l'usager.
  • Courriel Courriel de l'usager.
  • Mot de passe Le mot de passe de l'usager dans le système Karuta.
  • Créateur? Le compte utilisateur créé doit-il avoir les droits d'un designer Karuta? (En général, non!)

Désactiver utilisateur

Désactiver un compte utilisateur. Désactiver un compte plutôt que de le supprimer permet de conserver les partages aux portfolios et les associations aux groupes d'utilisateurs en prévision d'une réactivation ultérieure.

Champ à renseigner :

  • Login utilisateur Login du compte utilisateur. Ajouter un élément variable pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément valeur pour spécifier directement l'identifiant de l'utilisateur.

Activer utilisateur

Ré-activer un compte utilisateur précédement désactivé.

Champ à renseigner :

  • Login utilisateur Login du compte utilisateur. Ajouter un élément variable pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément valeur pour spécifier directement l'identifiant de l'utilisateur.

Supprimer utilisateur

Supprimer un compte utilisateur. (Attention : supprimer un utilisateur ne supprime pas ses portfolios!)

Champ à renseigner :

  • Login utilisateur Login du compte utilisateur. Ajouter un élément variable pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément valeur pour spécifier directement l'identifiant de l'utilisateur.

Créer arbre

Créer une instance de portfolio à partir d'un modèle de portfolio.

Champs à renseigner :

  • Référence arbre Attribuer un identifiant qui sera utilisé dans des commandes subséquentes pour faire référence à ce portfolio (par exemple pour le partager avec un utilisateur).
  • Modèle Code du modèle de portfolio à instancier. Format : code du projet.code du portfolio modèle. Ajouter des éléments variable pour des variables globales ou locales du fichier CSV et/ou des éléments valeur pour spécifier directement des valeurs.
  • Code Code de l'instance de portfolio qui sera créée. Format : code du projet.code de l'instance de portfolio. Ajouter des éléments variable pour des variables globales ou locales du fichier CSV et/ou des éléments valeur pour spécifier directement des valeurs.
  • Libellé Libellé de l'instance de portfolio qui sera créée. Ajouter des éléments variable pour des variables globales ou locales du fichier CSV et/ou des éléments valeur pour spécifier directement des valeurs.

  • Rôle utilisateur Rôle de l'utilisateur dans le portfolio (exemple : etudiant). Ajouter un élément variable pour puiser le rôle dans une variable globale ou locale du fichier CSV, ou un élément valeur pour spécifier directement le rôle de l'utilisateur.

  • Sélection proxy Ressource de type Proxy que l'on veut mettre à jour. Format : ID.tag sémantique ou #current_node. L'ID du portfolio a précédement été attribué soit dans un élément CreateTree soit dans un élément SelectTree, et le tag sémantique est celui de la ressource à mettre à jour dans le portfolio. #current_node est un mot réservé. Dans ce cas la ressource à mettre à jour est la ressource courante. Cela permet, par exemple, de mettre à jour une ressource qui vient d'être importée dans le portfolio.
  • Sélection source Noeud vers lequel pointe l'élément Proxy. Format : code portfolio.tag sémantique. Ajouter des éléments variable pour des variables globales ou locales du fichier CSV et/ou des éléments valeur pour spécifier directement des valeurs.
