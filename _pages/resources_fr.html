Les éléments de ressource contiennent les données d'un portfolio. Chaque élément ressource a une liste de droits (Voir, Éditer, Soumettre, etc). Pour chaque droit, le designer peut spécifier un ou plusieurs rôles (par exemple etudiant ou enseignant) qui auront ce droit. Il ou elle procède à cette spécification dans les métadonnées de l'élément. C'est le designer du portfolio qui décide quels seront les rôles pour les futurs utilisateurs du portfolio. Les utilisateurs seront ensuite partagés avec un portfolio en spécifiant un rôle.

Notez que le designer a tous les droits sur tous les éléments pendant qu'il construit le portfolio. Il est parfois utile au designer de pré-remplir les données dans un élément ressource afin de fournir des informations aux utilisateurs (par exemple fournir des instructions ou joindre un fichier au portfolio). Dans ce cas, on n'attribue généralement le droit d'édition à aucun rôle.

Dans la représentation XML du portfolio, les ressources sont les feuilles.

Les ressources disponibles sont :

  • Texte court
  • Texte long
  • Commentaires
  • Image
  • Audio
  • Vidéo
  • Document
  • oEmbed
  • URL
  • URL2Unit
  • Envoi de courriel

Texte court

L'élément Texte court est un champ de texte. Il est utilisé pour entrer une donnée courte comme un nom. Généralement on veut attribuer le droit d'Éditer la ressource à un rôle, par exemple etudiant ou enseignant.

Texte long

L'élément Texte long est utilisé pour entrer un texte plus long que le champ. L'utilisateur qui a le droit d'Éditer la ressource dispose d'un éditeur de texte simple qui offre des options de mise en forme de base.

Peut aussi être utilisé par le designer pour donner des instructions à l'utilisateur. Dans ce cas, le rôle Éditer la ressource n'est attribué à aucun utilisateur.

Commentaires

L'élément Commentaires est un élément Texte long qui est signé et daté. L'utilisateur qui édite le commentaire voit son nom d'utilisateur ajouté au bas de son commentaire, ainsi que l'heure et la date d'édition.

Habituellement utilisé avec un menu pour que l'utilisateur puisse ajouter des commentaires au besoin et un bouton soumettre pour pouvoir garder la trace de l'évolution des commentaires.

Image

L'élément Image permet de téléverser un fichier image (tous les formats courants sont acceptés). L'image est ensuite affichée dans le portfolio.

Il est possible d'attribuer au rôle qui a le droit d'Éditer la ressource un droit supplémentaire pour redéfinir la taille d'affichage de l'image (autrement la taille d'affichage est définie par le designer).

Audio

L'élément Audio permet de téléverser un fichier MP3. Cet élément vient avec un lecteur audio pour écouter le fichier téléversé.

Seul l'utilisateur avec le droit d'Éditer la ressource peut téléverser un fichier, mais tous les utilisateurs qui ont le droit de Voir l'élément (par défaut all, tous les rôles) peuvent l'écouter.

Vidéo

L'élément Vidéo permet de téléverser un fichier MP4. Cet élément vient avec un lecteur vidéo pour regarder le fichier téléversé.

Seul l'utilisateur avec le droit d'Éditer la ressource peut téléverser un fichier, mais tous les utilisateurs qui ont le droit de Voir l'élément peuvent le regarder.

Document

L'élément Document permet de téléverser un fichier de n'importe quel type au portfolio. Contrairement aux éléments Image, Audio et Vidéo, le contenu du fichier n'est pas affiché dans le portfolio — cependant le fichier est disponible pour téléchargement par tous les utilisateurs qui ont le droit de Voir l'élément.

oEmbed

L'élément oEmbed permet d'afficher dans le portfolio des présentations (vidéo, diapositives, etc) sauvegardées sur un site tiers (YouTube, Vimeo, Meetup, Facebook, etc). Le contenu désiré est spécifié par un hyperlien. Cet élément vient avec un lecteur qui permet de regarder la cible du lien oEmbed à même le portfolio.

Pour plus d'informations au sujet de l'élément oEmbed voir https://oembed.com/.

URL

L'élément URL permet de spécifier un hyperlien dans le portfolio. L'hyperlien doit rediriger vers une page à l'extérieur de Karuta — pour rediriger à l'intérieur de Karuta, voir l'élément URL2Unit (paragraphe suivant).

URL2Unit

L'élément URL2Unit permet de créer un lien vers une autre page du portfolio. Il peut être utilisé par le designer pour créer des liens de navigation dans le portfolio, autres que le menu de navigation à gauche, ou par l'utilisateur final pour spécifier des pages de son portfolio qu'il juge particulièrement importantes.

Envoi de courriel

Avec un élément Envoi de courriel, un usager peut envoyer un courriel à un destinataire pré-déterminé sans que l'adresse courriel du destinataire lui soit divulgée.

Calendrier

L'élément Calendrier permet de choisir une date dans un calendrier.

Couleur

L'élément Couleur permet de choisir une couleur dans une palette de couleur.

Tableau de bord

L'élément Tableau de bord exécute un script de rapport et affiche la table HTML résultante.

Le script de rapport est créé dans un portfolio séparé, le portfolio Rapport, et le code de ce portfolio est spécifié dans l'élément Tableau de bord. Le code du portfolio Rapport peut être suffixé d'une ou plusieurs des instructions suivantes :

  • @local Le script de rapport s'exécutera uniquement sur les données contenues dans la page courante (page qui contient l'élément Tableau de bord) plutôt sur tout le portfolio.
  • @nospinning La roulette d'attente (spinning wheel) n'est pas affichée.
  • @norefresh La table n'est pas mise à jour en cas de modification des données.

Rapport différé

Comme l'élément Tableau de bord, l'élément Rapport différé exécute un script de rapport et affiche la table HTML résultante. Cependant, au lieu d'être exécuté au moment d'afficher la page de portfolio, le script est exécuté à un moment et une fréquence pré-déterminé puis stocké en mémoire, et c'est le résultat pré-exécuté qui est affiché au moment de charger la page. Ceci est utile pour les scripts qui sont long à exécuter, comme les rapports de cohorte.

GetResource (liste déroulante)

L'élément GetResource permet de construire une liste d'éléments parmi laquelle l'utilisateur peut faire un choix (liste déroulante, boîte de choix ou encore radio-boutons).

Les éléments qui vont remplir la liste déroulante sont des éléments ressource de type Item (voir plus bas) ou des éléments Sous-section et ils sont habituellement créés dans un portfolio séparé : le portfolio Référentiel. Tous les éléments d'une liste déroulante doivent partager le même tag sémantique et chacun doit avoir un code unique (Code - Item dans le cas d'éléments Item et Code dans le cas d'éléments Sous-section).

On spécifie quels éléments du référentiel afficher dans la liste déroulante via la métadonnée Recherche de l'élément (voir la page MÉTADONNÉES pour plus de détails).

GetGetResource (liste déroulante conditionnelle)

L'élément GetGetResource permet de construire une liste déroulante dont les options sont restreintes par le choix fait par l'usager dans un élément GetResource associé. Le choix de l'utilisateur dans la première liste déroulante détermine quels seront les options offertes dans la deuxième.

On spécifie quels éléments du référentiel afficher dans la liste déroulante via la métadonnée Recherche de l'élément (voir la page MÉTADONNÉES pour plus de détails).

GetDoubleResource (double liste déroulante)

L'élément GetDoubleResource permet de construire deux boîtes indépendantes l'une de l'autre dans le même élément. Par exemple pour demander à l'utilisateur de choisir un semestre et une année.

Item

L'élément Item est un choix offert dans une liste déroulante. On regroupe les éléments Item d'un projet dans un portfolio référentiel. Tous les éléments d'une même liste déroulante doivent partager le même tag sémantique et chacun doit avoir un code unique.

Proxy

L'élément Proxy permet d'afficher un élément d'un portfolio dans un autre portfolio.

Il est possible de donner à l'usager le droit de modifier l'élément affiché par proxy, auquel cas la valeur de l'élément original est automatiquement mise à jour.

GetProxy

L'élément GetProxy permet de laisser l’utilisateur choisir la cible du Proxy affiché via une liste déroulante. Tous les éléments offerts dans la liste déroulante doivent patager le même tag sémantique (par exemple choisir parmi toutes les pages page-competence).
