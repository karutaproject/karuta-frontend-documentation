Resource elements contain the data of a portfolio. Each resource element has a list of rights (See, Edit, Submit, etc). For each right, the designer may specify one or multiple roles (for example student or teacher) that will have this right. This is done in the metadata of the element. It is the portfolio designer who decides what the various roles will be for the future portfolio users. Users will then be shared with a portfolio in a specific role.

Note that the designer has all the rights on all portfolio elements while building the portfolio. It is sometimes useful for the designer to pre-fill data in resource elements to supply information to users (for example give instructions or attach a file for the user to download). In this case, the Edit right is generally not attributed to any role.

In the XML representation of the tree, the resource elements are the leaves of the tree.

The available resources are:

  • Short Text
  • Long Text
  • Comments
  • Image
  • Audio
  • Video
  • Document
  • oEmbed
  • URL
  • URL2Unit
  • Send Email
  • Calendar
  • Color
  • Dashboard
  • Deferred Report
  • GetResource
  • GetGetResource
  • GetDoubleResource
  • Item
  • Proxy
  • GetProxy

Short Text

The Short Text element is a text field. It is used to enter short data, like a name. Usually the Edit right of the element will be attributed to a role, like student or teacher.

Long Text

The Long Text element is used to enter a longer text. A basic text editor with simple layout options is available to the user with the Edit right on this element.

This element is also useful to include instructions in the portfolio, from to designer to the users. In this case, the Edit right is not attributed to any role.

Comments

The Comments element is a Long Text which is signed and timestamped. The name of the user that edits the element, along with date and time, are added at the end of the comment.

Usually used in combination with a user menu that allows users to add comments as need be, and a submit button to keep track of the evolution of the comments.

Image

The Image element is used to upload an image file to the portoflio (all common formats are accepted). The image is then displayed in the portfolio.

It is possible to assign an extra right to the role that has the Edit Resource right to modify the display size (otherwise the display size is defined by the designer).

Audio

The Audio element is used to upload an MP3 file to the portfolio. A basic audio player is available to listen to the uploaded file.

Only the user with the Edit right may upload a file into this element, but any user with the See right (by default all, all roles), may listen to it.

Video

The Video element is used to upload an MP4 file to the portfolio. A basic video player is available to watch the uploaded file.

Only the user with the Edit right may upload a file into this element, but any user with the See right may watch it.

Document

The Document element is used to upload of a file of any type to the portfolio. Contrary to the Image, Audio and Video elements, the content of the file is not displayed directly in the portfolio — however the file is available for download by any user with the See right on the element.

oEmbed

The oEmbed element is used to display in the portfolio presentations (video, slides, etc) stored on a third party site (YouTube, Vimeo, Meetup, Facebook, etc). The desired content is specified by url. A basic player is available to view the content within the portfolio directly.

For more information on oEmbed elements, see https://oembed.com/.

URL

The URL element is used to specify a hyperlink in the portfolio. A label may also be specified, in which case it will be displayed on the link instead of the url. Any user with the See right on the element may follow the link.

The hyperlink must lead outside Karuta — for links within Karuta see the URL2Unit element below.

URL2Unit

The URL2Unit element is used to specify a hyperlink to another page of the portfolio. It may be used by the designer to create navigation links within the portfolio, other than the left-hand navigation menu, or it may be used by the user to indicate certain pages of their portfolio they consider particularly important.

Send Email

With a Send Email element a user may send an email to a predetermined recipient, without the recipient's email address being revealed to the user.

Calendar

The Calendar element is used to pick a date in a calendar.

Color

The Color element is used to pick a color in a color palette.

Dashboard

The Dashboard element executes a report script and displays the resulting HTML table.

The report script is created in a seperate portfolio, a Report portfolio, and the code of this portfolio is specified in the Dashboard element. The Report portfolio code may be suffixed by one or more of the following attributes:

  • @local The report script will execute only on the data of the current page (page containing the Dashboard element) rather than on the entire portolio.
  • @nospinning The spinning wheel is not displayed.
  • @norefresh The table is not updated if the data is modified.

Deferred Report

Like the Dashboard element, the Deferred Report element executes a report script and displays the resulting HTML table. However, instead of executing the script upon displaying the portfolio page, the script is executed according to a predetermined time and frequency and stored in memory, and the pre-executed result is displayed when loading the page. This is useful for scripts that are long to execute like aggregated cohort reports.

GetResource (choice box)

The GetResource element is used to build a list of elements amongst which the user can make a choice (dropdown list, choice box or radio button).

The elements that will fill the choice box are resource elements of type Item (see below) or Subsection elements and are usually created in a seperate portfolio: a Rubrics portfolio. All elements of a choice box must share the same semantic tag and each must have a unique code (Code - Item for Item elements and Code for Subsection elements).

The Search metadata of the GetResource element is used to specify which elements of the Rubrics portfolio will populate the dropdown list (see the METADATA page for more detail).

GetGetResource (conditional choice box)

The GetGetResource element is used to build a choice box whose options are restricted by the choice made by the user in an associated GetResource element. The user's selection in the first choice box determines which options will be displayed in the second.

The Search metadata of the GetGetResource element is used to specify which elements of the Rubrics portfolio will populate the choice box (see the METADATA page for more detail).

GetDoubleResource (double choice box)

The GetDoubleResource is used to build two independent choice boxes within one single element. For example to ask the user to pick a semester and a year.

Item

The Item element is one option of a choice box. A project's Item elements are grouped in a Rubrics portfolio. The code of each Item must be unique within the Rubrics portfolio (Code - Item). Items appearing in the same choice box must share the same semantic tag.

Proxy

The Proxy element is used to display an element of one portfolio in another portfolio.

It is possible to give the user the right to modify the element displayed by proxy, in which case the value of the original element is automatically updated.

GetProxy

The GetProxy element is used to let the user choose from a choice box the target of the displayed Proxy. All elements available in the choice box must share the same semantic tag (for example choose between all competency-page pages).
