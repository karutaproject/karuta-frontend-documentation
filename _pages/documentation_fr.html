KARUTA est un logiciel open source permettant de construire des portfolios. C'est un outil flexible qui permet de construire aussi bien des portfolios de présentation, que des portfolios d'apprentissage ou encore des portfolios d'évaluation.

Le développement de portfolios peut se faire à partir d'un modèle de portfolio vierge auquel on ajoute des éléments de base fournis par Karuta, ou encore à partir de templates fournis par la communauté puis modifiés pour répondre à des besoins spécifiques.

La présente documentation est organisée en trois sections :

  • Portfolio
  • Rapport
  • Batch

La section Portfolio présente les éléments de structure, les ressources (les éléments qui vont contenir les données), les métadonnées et la mise en forme (CSS).

La section Rapport présente les commandes de script de rapport. Ces commandes permettent de sélectionner des éléments d'un ou plusieurs portfolios et de les présenter sous forme de table HTML pour faire des tableaux de bord et des CV.

La section Batch présente les commandes de script de batch. Ces commandes permettent d'automatiser la création, modification et suppression de portfolios ainsi que la création et suppression d'utilisateurs.

Remarques d'ordre général

Les langues
KARUTA est multilingue (français et anglais par défaut). Les libellés des éléments de portfolio sont conservés dans la langue de travail active (on sélectionne la langue active dans le menu drapeau tout en haut des pages Karuta.) Les libellés dans les menus aux utilisateurs et dans les bulles d'aide sont suffixés par le code de la langue et séparés par une barre oblique (/).
Exemple : Ajouter une photo@fr/Add a picture@en.

Les partages et les rôles utilisateurs
Au moment du partage d'une instance de portfolio avec un utilisateur, cet utilisateur est associé au portfolio dans un rôle spécifique (par exemple, le rôle étudiant ou professeur). C'est le designer du portfolio qui choisi les noms des rôles. Pour chaque portfolio, Karuta créé un groupe d'utilisateurs pour chaque rôle rencontré dans le portfolio. Ensuite, pour associer un rôle à un utilisateur, Karuta place cet utilisateur dans le groupe du rôle correspondant.

Il y a trois rôles prédéfinis :

  • designer C'est l'utilisateur qui construit les portfolios. Il a tous les droits sur tous les éléments du portfolio, indépendamment des divers rôles utilisateurs qu'il spécifie dans les métadonnées d'un élément.
  • all Ce mot-clé inclu tous les rôles définis sur le portfolio. Par exemple, par défaut tous les utilisateurs ont le droit de voir tous les éléments d'un portfolio, mais ce comportement peut être modifié par le designer en retirant le rôle all dans les métadonnées.
  • user Ce mot-clé sera remplacé par l'identifiant de l'utilisateur lors de l'instantiation du portfolio. Cela devient donc un rôle unique puisqu'il n'y a qu'un seul utilisateur qui peut avoir ce rôle.
