The structural elements are used to organize the data in a portfolio.

In the XML representation of the portfolio, the structural elements are the branches of the tree, while the data elements (the resources) are the leaves of the tree.

The Karuta structural elements are:

  • Section
  • Page
  • Subsection

The Karuta data elements are:

  • Resource Node

Section

A Section element groups portfolio pages together. A Section may contain any other Karuta element: Sections, Pages, Subsections and Resource Nodes.

Page

A Page is the basic element through which portfolios are displayed. A Page may contain Subsections and Resource Nodes.

Subsection

A Subsection element groups Resource Nodes together. A Subsection may contain other Subsections and Resource Nodes.

Resource Node

A Resource Node is the terminating element of a tree (the leaves). A resource is not a structural element, it contain the data of the portfolio. There are multiple types of resources: text field, text editor, file, image, audio and video player, calendar, etc (see the RESOURCES page for more detail).
